# Цель проекта
Разработать отчет по пожертвованиям благотворительного фонда, где были бы отображены успешные пожертвования за период 18.11.2025-26.11.2025. 
### Примечание 
Тренировочные данные были предоставлены благотворительным фондом в качестве тестового задания.

оно было доработано и специально выполнено в PowerQuery с последующей визуализацией в PowerBI, чтобы показать владение этими инструментами.
## Описание проекта
### Лист checks
Используйте лист payments_raw. Это таблица с пожертвованиями, в которой есть ошибки и некорректные данные. 

Подсчет базовых показателей На новом листе с названием checks укажите следующие показатели: 
- Общее количество строк в таблице. 
- Количество платежей со статусом success. 
- Количество платежей со статусом refund. 
- Количество платежей со статусом fail. 

На том же листе checks укажите: 
- Количество строк, где поле donor_email пустое. 
- Количество строк, где email выглядит некорректно.
- Количество строк, где поле donor_name пустое или содержит только один символ. 
- Количество дублирующихся payment_id 


Что нужно учесть: 
- Все расчеты должны быть сделаны с помощью формул, а не вручную. 
- Результаты должны автоматически обновляться при изменении данных.
### Лист daily_report
Что нужно сделать: Используя данные на листе payments_raw, необходимо сделать ежедневный отчет по пожертвованиям. Создайте новый лист с названием: daily_report 
Как должен выглядеть отчет В строках: 
- дата платежа (без времени)

В колонках: 
- date — дата платежа 
- Total_success_amount — сумма всех успешных пожертвований за день
- Success_count — количество успешных платежей за день
- Unique_donors_count — количество уникальных благотворителей за день (по email)
- IMZ_amount — сумма пожертвований с назначением IMZ
- SCIENCE_amount — сумма пожертвований с назначением SCIENCE
- GENERAL_amount — сумма всех остальных пожертвований

Что нужно учесть: 
- В отчет включайте только платежи со статусом success.
- Для расчета сумм по назначению используйте колонку purpose_code.
- Если purpose_code не равен IMZ или SCI, считайте его как GENERAL.
- Пустые email не учитывайте при подсчете уникальных благотворителей.
- Расчеты должны быть автоматическими, а не посчитанные вручную


### Визуализация PowerBI 
Необходимо: 
- Показать общую сумму пожертвований, количество уникальных благотворителей, среднюю сумму пожертвований за день. 
- Показать сумму пожертвования в динамике.
- Сравнить пожертвования в зависимости от назначения.
